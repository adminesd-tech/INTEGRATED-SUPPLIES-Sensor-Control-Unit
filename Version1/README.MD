# SENSOR CONTROL UNIT (SCU-1) — User Manual

คู่มือการใช้งานโปรแกรม SCU-1 สำหรับตั้งค่า Sensor Limit ผ่าน Serial

---

## 🔷 1. Overview

SCU-1 เป็นโปรแกรมสำหรับ:

* เชื่อมต่อกับตัวเครื่องผ่าน USB
* อ่านค่า limit ปัจจุบันจากระบบ
* ปรับตั้งค่า Sensor (CH01 / CH02)
* ส่งค่ากลับไปยังอุปกรณ์

---

## 🔷 2. การติดตั้ง

### กรณีใช้งาน .exe

1. เปิดไฟล์:

   ```
   SCU1.exe
   ```
2. ถ้ามีแจ้งเตือนจาก Windows:

   * กด **More info**
   * กด **Run anyway**

---

## 🔷 3. การเชื่อมต่ออุปกรณ์

### ขั้นตอน

1. เสียบตัวเครื่องเข้าคอม
2. เปิดโปรแกรม SCU-1
3. เลือก COM port จาก dropdown
4. โปรแกรมจะ **connect อัตโนมัติ**

---

## 🔷 4. สถานะระบบ

| สถานะ      | ความหมาย           |
| ---------- | ------------------ |
| CONNECTED  | เชื่อมต่อสำเร็จ    |
| OFFLINE    | ยังไม่เชื่อมต่อ    |
| CONNECTING | กำลังเชื่อมต่อ     |
| FAILED     | เชื่อมต่อไม่สำเร็จ |

---

## 🔷 5. การอ่านค่า Sensor (Auto)

หลังจาก connect สำเร็จ:

* โปรแกรมจะอ่านค่า limit จากบอร์ดอัตโนมัติ
* ค่า CH01 และ CH02 จะถูกแสดงบนหน้าจอ

### ตัวอย่าง

```
CH01 = 30
CH02 = 100
```

---

## 🔷 6. การตั้งค่า Limit

### ขั้นตอน

1. ใส่ค่าที่ต้องการใน:

   * CH01 (Sensor ชิ้นงาน)
   * CH02 (Sensor พนักงาน)

2. กดปุ่ม:

   ```
   WRITE CONFIGURATION
   ```

3. รอข้อความ:

   ```
   ✓ WRITTEN
   ```

---

## 🔷 7. กฎของค่า Limit

* ค่า CH01 และ CH02 ต้องเป็นตัวเลข
* ต้องมากกว่า 3 และน้อยกว่า 450 cm
* แนะนำ:

  ```
  CH01 < CH02
  ```

---

## 🔷 8. การแสดงผล

### ด้านบน

* แสดงชื่อระบบ + เวลา

### ตรงกลาง

* ค่า Sensor 2 ช่อง:

  * CH01 (PRODUCT)
  * CH02 (EMPLOYEE)

### ด้านล่าง

* แสดงสถานะ:

  ```
  CONNECTED · COM7 · 9600 BAUD
  ```

---

## 🔷 9. กรณีผิดพลาด

### ❌ ไม่มี COM port

* กดปุ่ม **SCAN**
* ตรวจสอบสาย USB

---

### ❌ Connect ไม่ได้

* ตรวจสอบ driver USB
* ตรวจสอบ port ว่าถูกต้อง

---

### ❌ ค่าไม่ขึ้น (0,0)

* บอร์ดอาจไม่ตอบกลับ
* ตรวจสอบ firmware
* ลองเสียบใหม่

---

### ❌ Write ไม่สำเร็จ

* ตรวจสอบว่า connect อยู่หรือไม่
* ตรวจสอบค่าที่ใส่

---

## 🔷 10. Tips การใช้งาน

* ใช้สาย USB คุณภาพดี
* หลีกเลี่ยงการเสียบ/ถอดระหว่างใช้งาน
* ถ้าโปรแกรมค้าง ให้ปิดแล้วเปิดใหม่

---

## 🔷 11. ข้อกำหนดระบบ

* Windows OS
* USB Port
* Driver USB-to-Serial (CH340 / CP210x)

---

## 🔷 12. FAQ

### Q: ทำไมเปิดโปรแกรมแล้วไม่เห็น COM?

A: อาจยังไม่ได้เสียบอุปกรณ์ หรือสาย USB ไม่เสถียร

---

### Q: ทำไมค่าเป็น 0,0?

A: ยังไม่ได้ connect หรือ connect ผิด COM Port

---

### Q: ใช้กับ Mac ได้ไหม?

A: ไม่ได้ Software Version1 (v1) รองรับการทำงานกับระบบปฏิบัติการ Window
